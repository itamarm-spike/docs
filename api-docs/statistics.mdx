This document provides an overview of the different statistics types available in our API, how they are calculated, and how you can query them using our RESTful API.


### Querying Statistics

To query statistics, you can use our RESTful API. Below is an example of how to query for statistics:

#### Request Example

```json
{
  "from": "2024-01-01",
  "to": "2024-01-02",
  "types": ["steps"],
  "interval": "day"
}
```

<ParamField body="from" type="string" required>
  The start date for the statistics query.
</ParamField>

<ParamField body="to" type="string" required>
  The end date for the statistics query.
</ParamField>

<ParamField body="types" type="array" required>
  An array of statistic types you want to query.
For a detailed overview of all available statistics, including their methods, metrics, units, and descriptions, please refer to the [Statistics Support Matrix](/statistics_matrix).
</ParamField>

<ParamField body="interval" type="string" required>
  The interval window for aggregating data (e.g., `day`, `hour`).
</ParamField>

<ParamField body="include_record_ids" type="boolean">
  Whether to include the record IDs in the response.
</ParamField>

<ParamField body="include_providers" type="boolean">
  Whether to include the relevant provider slug in the response.
</ParamField>


#### Response Example

```json
{
  "statistics": [
    {
      "timestamp_from": "2024-01-01",
      "timestamp_to": "2024-01-02",
      "values": {
        "steps": 1000
      },
      "record_ids": ["ff000000-0000-0000-0000-000000000000"]
    }
  ]
}
```

<ResponseField name="timestamp_from" type="string" required>
  The start timestamp of the aggregated data.
</ResponseField>

<ResponseField name="timestamp_to" type="string" required>
  The end timestamp of the aggregated data.
</ResponseField>

<ResponseField name="values" type="object" required>
  The aggregated values for the specified statistic types.
</ResponseField>

<ResponseField name="record_ids" type="array">
  Unique identifiers for the records included in the aggregation. Would only
  be present if the `include_record_ids` parameter is set to `true`.
</ResponseField>

<ResponseField name="providers" type="array">
  The relevant provider slug for the records included in the aggregation. Would
  only be present if the `include_providers` parameter is set to `true`.
</ResponseField>

<Note>
